# Code Of Conduct

GradleMC welcomes useful reports, focused contributions, and honest technical disagreement.

It does not welcome harassment, spam, personal attacks, entitlement, fake support claims, or drive-by chaos pretending to be collaboration.

---

## Expected Behavior

- Be direct, specific, and evidence-based.
- Criticize ideas, code, docs, claims, and workflows without attacking people.
- Use issue templates and include reproduction details.
- Respect license boundaries and third-party work.
- Keep support requests focused on GradleMC.
- Review logs and reports before posting private information.
- Assume unsupported loaders and versions are unsupported until the project says otherwise.

---

## Unacceptable Behavior

- Harassment, abuse, threats, slurs, or targeted insults.
- Posting private information without consent.
- Spamming issues, pull requests, or comments.
- Reopening the same rejected request without new evidence.
- Uploading copied assets, mod jars, private files, or unlicensed material.
- Pushing fake support claims for NeoForge, Quilt, Bedrock, or other unsupported Minecraft targets.
- Demanding urgent support without logs, reproduction steps, or basic details.

---

## Maintainer Actions

Maintainers may:

- edit issue titles for clarity;
- request logs, reproduction steps, or screenshots;
- close duplicates, unsupported requests, vague reports, or spam;
- lock heated or unproductive threads;
- block users who repeatedly create hostile, unsafe, or useless noise.

This is not bureaucracy. It is pest control for project quality.

---

## Reporting Problems

Use GitHub issues for public behavior problems that are safe to discuss publicly.

Do not post sensitive personal data, private server details, tokens, or credentials. Redact first.
