GradleMC

In-game diagnostics, stability checks, and exportable troubleshooting reports for Minecraft Forge modpacks.

GradleMC helps players, pack makers, server owners, and testers understand what is happening inside a modded Minecraft instance before troubleshooting turns into blind guessing.

This page is for GradleMC v1.0.1, built for Minecraft Java Edition 1.20.1 on Forge.

What GradleMC does

  • Diagnostics GUI: open the in-game control center with /gradlemc gui or the configurable GUI keybind.
  • Readable commands: use lowercase /gradlemc commands for status, memory, checks, reports, performance samples, Smart Diagnostics, and more.
  • Exportable reports: write local troubleshooting reports under <gameDir>/gradlemc/reports/.
  • Mod and config inspection: view loaded mod details, config paths, report paths, local rule checks, and environment context.
  • Bounded performance checks: sample TPS/MSPT, FPS, entity density, block entity density, and passive worldgen pressure within configured limits.
  • Profiler foundation: capture bounded tick, CPU-lite, memory-lite, and combined local profiling summaries.
  • Smart Diagnostics: get local rule-based scoring, advice, evidence, confidence, trend notes, and missing-data notes.
  • Adaptive diagnostics: use lightweight local gameplay-state logic for bounded adaptive status and warnings.

Quick start

  1. Install Minecraft Java Edition 1.20.1.
  2. Install Forge 47.4.20 or a compatible Forge 47.x setup.
  3. Use Java 17.
  4. Put gradlemc-1.0.1-forge-1.20.1.jar in the client or server mods folder.

Install on the client for the GUI, keybind, overlay, and FPS testing. Install on the server for server-side commands, reports, TPS/MSPT sampling, passive worldgen observation, scans, issue bundles, Smart Diagnostics, and adaptive diagnostics state.

Useful commands

  • /gradlemc gui - open the diagnostics GUI.
  • /gradlemc status - show current GradleMC status and paths.
  • /gradlemc version - show GradleMC, Minecraft, Forge, Java, and loaded-mod context.
  • /gradlemc memory - show JVM heap information.
  • /gradlemc check - run stability checks.
  • /gradlemc export - write a troubleshooting report.
  • /gradlemc smart score - calculate a local Smart Diagnostics score.
  • /gradlemc smart advice - show prioritized local recommendations.

Privacy and honesty

  • No telemetry.
  • No analytics.
  • No cloud AI.
  • No LLM or generative AI integration.
  • No hidden online inference.
  • No broad private-file scans by default.

Reports can still include local paths, mod names, Java details, and runtime context. Review exports before sharing them publicly.

Current support limits

This release supports Forge for Minecraft Java Edition 1.20.1.

Fabric, NeoForge, Quilt, Bedrock, and other Minecraft versions are not included in this artifact. Future ports are roadmap work until the code, build, runtime checks, docs, and artifact naming all agree.

Best use case

GradleMC is best used when you need clearer evidence for crash, lag, support, or modpack stability conversations. It will not magically fix a broken pack, but it gives you better local information so you can stop guessing and start debugging with actual data.
