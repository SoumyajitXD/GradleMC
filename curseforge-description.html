What GradleMC does

GradleMC helps players, modpack creators, server owners, testers and developers understand a modded instance before troubleshooting degenerates into random mod deletion.

It exposes useful environment and runtime information through an in-game interface, lowercase /gradlemc commands and local files. It can highlight suspicious pressure and produce reviewable evidence. It does not automatically repair a pack, bypass hardware limits or manufacture frames from optimism.

A practical workflow

01 — Inspect

Check the active Minecraft release, loader, Java runtime, GradleMC build, loaded mods, important paths, configuration state and surrounding runtime context.

02 — Measure

Run bounded observations for FPS, server performance, TPS/MSPT, memory pressure, entities, block entities, world generation and lightweight profiling where the installed build supports them.

03 — Interpret

Use local rule-based Smart Diagnostics to connect available signals, show missing evidence, calculate scores and present prioritized recommendations with context.

04 — Export

Generate reports, profiler summaries and issue bundles for support requests, bug reports, test comparisons and modpack maintenance.

Core capabilities

Inspect the instance

  • Minecraft, loader, Java and GradleMC build information.
  • Loaded-mod summaries, counts, searches and inspection tools.
  • Configuration, report and other relevant local paths.
  • Memory state and runtime context.

Measure runtime pressure

  • Client FPS and timed FPS testing where client support is installed.
  • Server performance observations including TPS/MSPT-related evidence.
  • Memory, entity, block-entity and world-generation pressure signals.
  • Bounded tick, CPU-lite, memory-lite and combined profiler modes with TXT and JSON output.

Interpret available evidence

  • Local rule-based Smart Diagnostics scores and advice.
  • Supporting evidence, confidence context, trends and missing-data warnings.
  • Adaptive local thresholds and observations without cloud inference.

Export something support can use

  • Readable diagnostic reports and recent-report discovery.
  • Profiler summaries and local profile files.
  • Reviewable issue bundles for focused troubleshooting.

The profiler is deliberately bounded and honest about its scope; it is not async-profiler or an allocation profiler.

Client and server roles

Install on the client for

The diagnostics GUI, configurable keybind, optional overlay, current and average FPS information, timed FPS tests and client-side environment context.

Install on the server for

Server command execution, permission-controlled tests, TPS/MSPT observations, memory and environment reports, loaded-mod inspection, world-generation observations, profiling, issue bundles and Smart Diagnostics summaries.

Install GradleMC on both sides when the investigation needs both client and server evidence. Client-only features cannot run from a dedicated-server console, because a console is not secretly a Minecraft window.

Installation

  1. Choose your exact Minecraft version and loader.
  2. Use the Java version listed for that Minecraft release line.
  3. Download the matching GradleMC jar from the CurseForge Files page.
  4. Place it in the instance or server mods folder.
  5. Launch the client or server.
  6. Run /gradlemc to inspect the command tree provided by that build.

Essential commands

Command literals are lowercase. Permission requirements and side availability still apply.

  • /gradlemc — show the installed build's command help.
  • /gradlemc gui — open the in-game interface for a player.
  • /gradlemc status — show a compact diagnostics summary.
  • /gradlemc version — show GradleMC, Minecraft, loader and Java context.
  • /gradlemc memory — inspect JVM memory information.
  • /gradlemc check — run local checks.
  • /gradlemc testfps start <seconds> and stop — run or stop a bounded client FPS test.
  • /gradlemc perf start <seconds> and stop — run or stop a bounded server performance sample.
  • /gradlemc export — write local troubleshooting evidence.
  • /gradlemc reports latest — locate the newest report.
  • /gradlemc smart score and /gradlemc smart advice — inspect local rule-based findings.

Additional commands exist for profiling, world-generation observation, mods, entities, reports, issue bundles, rules and other diagnostics. Use /gradlemc rather than treating an old web page as immutable scripture.

Reports and generated files

GradleMC writes its local data beneath:

<gameDir>/gradlemc/

Depending on the build and action used, this can include reports, exports, issue bundles, profiler output, rules and supporting diagnostic data.

Review before sharing. Generated evidence can contain local paths, Java information, mod names, loader details, instance information and system context. GradleMC does not automatically upload these files; publishing them is your decision.

Local and privacy-first

  • No behavioural telemetry or analytics.
  • No cloud AI, LLM integration or hidden remote inference.
  • No automatic report upload.
  • No broad scan of unrelated private files.
  • Generated evidence remains local until you choose to share it.

Smart Diagnostics is not generative AI. It is a local rule-based system that evaluates the evidence GradleMC has collected, identifies missing information and produces scored, prioritized guidance.

Honest limitations

  • Not a crash-fixing bot: it can expose evidence, not automatically repair every pack.
  • Not an optimization mod: it measures performance; it does not replace Sodium, Embeddium or careful configuration.
  • Not a deep-profiler replacement: difficult CPU, allocation or tick investigations may still require Spark, VisualVM, async-profiler or similar specialist tools.
  • Not a universal jar: each loader and Minecraft release needs its own matching file.
  • Not a Bedrock tool: GradleMC targets the listed Minecraft Java Edition releases.
