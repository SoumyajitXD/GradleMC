The GradleMC diagnostics control center

Debug with evidence, not vibes

GradleMC helps players, modpack creators, server owners, developers, and testers understand what is happening inside a modded Minecraft instance before troubleshooting becomes random mod deletion.

It combines environment inspection, memory diagnostics, bounded performance testing, runtime observations, local stability checks, Smart Diagnostics, profiling summaries, reports, and issue-bundle exports inside one readable tool.

GradleMC does not pretend to repair every broken pack automatically. It gives you the evidence needed to identify the actual problem instead of sacrificing random mods to the troubleshooting gods.

A practical diagnostic workflow

01 — Inspect

Review the Minecraft version, mod loader, Java runtime, GradleMC build, loaded mods, memory state, configuration paths, report paths, and relevant runtime context.

02 — Measure

Run bounded FPS, TPS/MSPT, memory, entity-density, block-entity, performance, profiler, and world-generation observations without leaving tests running forever.

03 — Interpret

Use local rule-based diagnostics to connect available signals, identify missing evidence, highlight suspicious pressure, and prioritize reasonable next steps.

04 — Export

Generate readable reports and reviewable issue bundles for support requests, bug reports, modpack testing, and comparisons between runs.

What GradleMC includes

Diagnostics control center

An in-game GUI for status information, quick actions, tests, profiler tools, adaptive diagnostics, reports, settings, and build information.

Readable command tree

Lowercase /gradlemc commands for inspection, testing, reporting, exports, memory information, and Smart Diagnostics.

Environment and mod inspection

Inspect the active Minecraft release, loader, Java runtime, loaded mods, GradleMC version, important directories, and configuration state.

Memory diagnostics

Review JVM heap use, available memory, allocation pressure, observed trends, and the surrounding runtime context.

Performance sampling

Collect bounded FPS, average FPS, TPS/MSPT, entity-density, block-entity, memory, profiler, and world-pressure observations.

Lightweight profiler foundation

Produce focused tick, memory, and combined TXT or JSON summaries without pretending to replace a dedicated deep profiler.

Smart Diagnostics

Receive local rule-based scoring, prioritized recommendations, supporting evidence, confidence notes, trend information, and warnings when evidence is incomplete.

Adaptive local diagnostics

Observe lightweight gameplay and runtime signals locally without telemetry, remote analytics, an external AI service, or hidden cloud inference.

Reports and issue bundles

Generate organized troubleshooting evidence that can be reviewed before it is shared with pack developers, server staff, mod authors, or support communities.

Configurable performance information without permanently covering the screen in dashboard soup

Installation

  1. Choose the required Minecraft release and mod loader.
  2. Install the Java version required by that Minecraft release line.
  3. Download the matching GradleMC file from CurseForge.
  4. Place the jar inside the instance or server mods folder.
  5. Launch Minecraft or start the server.
  6. Run /gradlemc gui or /gradlemc status.

Install on the client for

The diagnostics GUI, configurable keybind, client overlay, FPS monitoring, timed FPS tests, and local client-environment information.

Install on the server for

Server commands, TPS/MSPT observations, memory and environment reports, loaded-mod inspection, passive world-generation observation, issue bundles, and Smart Diagnostics summaries.

Useful commands

/gradlemc gui
Open the GradleMC diagnostics interface.

/gradlemc status
Show current GradleMC status, environment details, and output locations.

/gradlemc version
Display the GradleMC, Minecraft, loader, and Java context.

/gradlemc memory
Inspect JVM heap use and memory-pressure information.

/gradlemc check
Run local environment and stability checks.

/gradlemc testfps start <seconds>
Start a bounded FPS test for the requested duration.

/gradlemc testfps stop
Stop the active FPS test.

/gradlemc perf start <seconds>
Run a bounded runtime-performance test.

/gradlemc export
Generate local troubleshooting evidence.

/gradlemc reports latest
Locate the newest generated GradleMC report.

/gradlemc smart score
Calculate a local Smart Diagnostics score from available evidence.

/gradlemc smart advice
Show prioritized local recommendations and their supporting signals.

Commands are lowercase: use /gradlemc, not /GradleMC. Exact subcommand availability may vary slightly between release lines; run /gradlemc to inspect the command tree provided by your installed build.

Reports and generated files

GradleMC stores generated output beneath:

<gameDir>/gradlemc/

Depending on the installed release and feature used, this directory can contain reports, exports, issue bundles, profiler summaries, diagnostic rules, and related GradleMC data.

Review before sharing. Reports may contain local paths, Java details, mod names, loader information, system context, and other troubleshooting evidence. GradleMC keeps the data local, but posting a report publicly is still your decision.

Local and privacy-first by design

✓ No telemetry
✓ No behavioural analytics
✓ No cloud AI service
✓ No LLM integration
✓ No hidden remote inference
✓ No broad private-file scanning

Smart Diagnostics is not generative AI. It is a local rule-based diagnostic system that evaluates available evidence, notes missing information, calculates diagnostic scores, and produces prioritized advice.

What GradleMC is not

  • Not a crash-fixing bot: it highlights evidence and suspicious signals but cannot automatically repair every broken pack.
  • Not a performance-boosting mod: GradleMC measures and explains performance; it does not manufacture extra FPS.
  • Not a deep-profiler replacement: advanced CPU, allocation, or tick investigations may still require specialist tools.
  • Not cross-loader magic: Forge, Fabric, NeoForge, and Quilt files are separate and are not interchangeable.
  • Not Bedrock software: GradleMC is made for the listed Minecraft Java Edition releases.
  • Not the Gradle build system: GradleMC is an independent Minecraft project and does not replace or represent Gradle.

Frequently asked questions

Does GradleMC automatically fix lag or crashes?

No. It collects evidence, highlights suspicious signals, and makes your next troubleshooting decision less blind.

Is GradleMC itself an FPS optimization mod?

No. It is a diagnostic tool. It can help reveal performance pressure, but it does not replace optimization mods, configuration work, or hardware limits.

Does Smart Diagnostics send anything online?

No. Smart Diagnostics and adaptive diagnostics work locally without an LLM, cloud AI, telemetry, or behavioural analytics.

Should I install GradleMC on the client or server?

Install it on the client for the GUI, keybind, overlay, and FPS tools. Install it on the server for server commands, TPS/MSPT observations, reports, world-generation observation, and server-side diagnostics. You may install it on both when investigating both sides.

Can I use a Forge file on NeoForge, Fabric, or Quilt?

No. Download the separate GradleMC file matching the exact loader and Minecraft version.

Where are generated reports stored?

Look beneath <gameDir>/gradlemc/ or use /gradlemc reports latest.
