What GradleMC is

GradleMC is a practical diagnostics control center for modded Minecraft. It gives you readable in-game commands, a diagnostics GUI, memory checks, loader and mod inspection, performance samples, Smart Diagnostics, issue bundles, and local reports you can actually use.

It does not pretend to magically fix your pack. It gives you evidence so you can stop debugging by vibes like a caveman with a config folder.

Quick start

  1. Use Minecraft Java Edition 1.20.1.
  2. Use Java 17.
  3. Download the GradleMC file that matches your loader.
  4. Put the jar into your client or server mods folder.
  5. Launch the game or server.
  6. Run /gradlemc gui or /gradlemc status.

Install GradleMC on the client for the GUI, keybind, overlay, and FPS sampling. Install it on the server for commands, reports, TPS/MSPT sampling, passive worldgen observation, scans, issue bundles, Smart Diagnostics, and adaptive diagnostics state.

Useful commands

/gradlemc gui Open the diagnostics GUI.
/gradlemc status Show current GradleMC status and paths.
/gradlemc version Show Minecraft, loader, Java, GradleMC, and mod context.
/gradlemc memory Inspect JVM heap usage and memory pressure.
/gradlemc check Run local stability checks.
/gradlemc export Write a local troubleshooting report.
/gradlemc reports latest Find the newest generated report.
/gradlemc smart score Calculate a local Smart Diagnostics score.
/gradlemc smart advice Show prioritized local recommendations.

Command rule: Minecraft commands are lowercase. Use /gradlemc, not /GradleMC.

What GradleMC checks

  • Minecraft, loader, Java, GradleMC, and loaded-mod details.
  • JVM heap pressure and memory status.
  • Config and report directory access.
  • Loaded mod count, previews, and mod ID search.
  • Entity and block entity density signals.
  • Bounded TPS/MSPT, FPS, and profiler samples.
  • Passive chunk and worldgen pressure observations.
  • Smart Diagnostics score, evidence, confidence, and missing-data notes.
  • Issue bundles designed for review before sharing.

Generated output is written under <gameDir>/gradlemc/, including reports, exports, issue bundles, profiles, and rules.

Privacy and honesty

No telemetry No analytics No cloud AI
No LLM integration No hidden inference No broad private-file scans

Smart Diagnostics and adaptive diagnostics are local rule-based systems. They are not cloud AI, LLMs, generative AI, telemetry, analytics, or online inference.

Reports can still include local paths, mod names, Java details, loader details, and runtime context. Review exports before posting them publicly. Evidence is powerful; leaking your own paths is just free clown makeup.

Current support limits

Supported public target: Minecraft Java Edition 1.20.1. GradleMC currently has public Forge, Fabric, and Quilt builds for 1.20.1.

Not currently claimed here: NeoForge, Bedrock, or future Minecraft versions unless a separate matching GradleMC file exists and the release page says so.

Best use case

Use GradleMC when a pack is crashing, lagging, acting unstable, or producing support threads where everyone is guessing and nobody has evidence. It gives you cleaner local information for crash, lag, support, and modpack stability debugging.

Less guessing. More evidence. Better troubleshooting.
