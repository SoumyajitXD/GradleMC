What GradleMC does

GradleMC is a practical diagnostics control center for modded Minecraft. It helps players, pack makers, server owners, and testers understand what is happening inside an instance: memory pressure, loaded mods, reports, paths, performance samples, worldgen pressure, Smart Diagnostics, issue bundles, and local troubleshooting evidence.

It does not magically fix broken packs. That would be witchcraft with a progress bar. GradleMC gives you cleaner information so you can debug with evidence instead of vibes.

Core features

  • Diagnostics GUI: open the in-game control center with /gradlemc gui or the GradleMC keybind.
  • Readable command tree: use lowercase /gradlemc commands for status, memory, checks, exports, reports, and diagnostics.
  • Exportable reports: generate local troubleshooting reports under <gameDir>/gradlemc/.
  • Mod and environment inspection: view Minecraft, loader, Java, GradleMC, loaded-mod, config, and path details.
  • Memory diagnostics: inspect JVM heap usage and memory-pressure context.
  • Bounded performance checks: sample TPS/MSPT, FPS, entity density, block entity density, and passive worldgen pressure.
  • Profiler foundation: collect bounded tick, CPU-lite, memory-lite, and combined summaries with local output.
  • Smart Diagnostics: get local rule-based scoring, advice, evidence, confidence, trends, and missing-data notes.
  • Adaptive diagnostics: lightweight local gameplay-state logic for bounded adaptive status and warnings.
  • Issue bundles: create support bundles designed for review before sharing.

Quick start

  1. Install Minecraft Java Edition 1.20.1.
  2. Use Java 17.
  3. Pick the correct loader file:
    • Forge: use the Forge 1.20.1 GradleMC file.
    • Fabric: use the Fabric 1.20.1 GradleMC file.
  4. Put the matching GradleMC jar into the client or server mods folder.
  5. Launch the game/server and use /gradlemc gui or /gradlemc status.

Install it on the client for the GUI, keybind, overlay, and FPS testing. Install it on the server for commands, reports, TPS/MSPT sampling, passive worldgen observation, scans, issue bundles, Smart Diagnostics, and adaptive diagnostics state.

Useful commands

/gradlemc gui Open the diagnostics GUI.
/gradlemc status Show current GradleMC status and paths.
/gradlemc version Show GradleMC, Minecraft, loader, Java, and loaded-mod context.
/gradlemc memory Show JVM heap information.
/gradlemc check Run local stability checks.
/gradlemc export Write a troubleshooting report.
/gradlemc smart score Calculate a local Smart Diagnostics score.
/gradlemc smart advice Show prioritized local recommendations.

Command rule: Minecraft commands are lowercase. /GradleMC is not fancy. It is just wrong.

Privacy and honesty

  • No telemetry.
  • No analytics.
  • No cloud AI.
  • No LLM or generative AI integration.
  • No hidden online inference.
  • No broad private-file scans by default.

Smart Diagnostics and adaptive diagnostics are local rule-based systems. They are not ChatGPT taped to a Minecraft jar with duct tape and delusion.

Reports can still include local paths, mod names, Java details, loader details, and runtime context. Review exports before posting them publicly.

Current support limits

Supported public target: Minecraft Java Edition 1.20.1. GradleMC currently has public Forge and Fabric builds for 1.20.1.

Not currently claimed here: NeoForge, Quilt, Bedrock, or future Minecraft versions unless a separate matching GradleMC file exists and the release page says so. Unsupported ports belong on the roadmap, not in marketing cosplay.

Best use case

Use GradleMC when a pack is crashing, lagging, acting unstable, or generating terrible support conversations. It gives you cleaner local evidence for crash, lag, support, and modpack stability debugging.

Less guessing. More evidence. Fewer support threads that look like crime scenes.
