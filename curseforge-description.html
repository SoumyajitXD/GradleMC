Debug with evidence, not vibes

GradleMC helps players, modpack creators, server owners, and testers understand what is happening inside a modded Minecraft instance.

It brings environment inspection, memory diagnostics, bounded performance sampling, local stability checks, Smart Diagnostics, report generation, and issue-bundle exports into one readable interface.

GradleMC does not magically repair broken packs. It gives you better evidence so you can find the actual problem instead of deleting random mods until the game reluctantly opens.

1. Inspect

See your Minecraft version, loader, Java runtime, GradleMC build, loaded mods, paths, configuration state, memory use, and runtime context.

2. Measure

Run bounded FPS, TPS/MSPT, memory, entity-density, block-entity, profiler, and world-generation observations.

3. Export

Generate readable reports and reviewable issue bundles that give support conversations something more useful than “it lags.”

Inspect Minecraft, loader, Java, GradleMC, loaded mods, configuration paths, report paths, and runtime details.		 Memory diagnostics

Performance sampling

Exportable reports

Installation

  1. Choose your Minecraft version and loader from the release table.
  2. Install the required Java version: Java 17, Java 21, or Java 25 depending on the Minecraft release line.
  3. Download the GradleMC jar matching both your Minecraft version and loader.
  4. Place the jar inside the instance or server mods folder.
  5. Launch Minecraft or start the server.
  6. Open GradleMC with /gradlemc gui or run /gradlemc status.

Client and server roles

Install on the client for:
  • Diagnostics GUI
  • Configurable GradleMC keybind
  • Client overlay features
  • Client FPS sampling
  • Local client environment information
 Install on the server for:
  • Server commands and reports
  • TPS and MSPT sampling
  • Memory and environment checks
  • Passive world-generation observation
  • Issue bundles and Smart Diagnostics summaries

Useful commands

/gradlemc gui Open the diagnostics GUI.
/gradlemc status Show GradleMC status, environment information, and output paths.
/gradlemc version Display Minecraft, loader, Java, GradleMC, and mod context.
/gradlemc memory Inspect JVM heap usage and memory-pressure information.
/gradlemc check Run local stability and environment checks.
/gradlemc export Generate a local troubleshooting report.
/gradlemc reports latest Locate the newest generated GradleMC report.
/gradlemc smart score Calculate a local Smart Diagnostics score.
/gradlemc smart advice Show prioritized local recommendations and supporting evidence.

Commands are lowercase: use /gradlemc, not /GradleMC.

Reports and generated files

GradleMC writes its generated output beneath:

<gameDir>/gradlemc/

Depending on the feature and release, this directory can contain reports, exports, issue bundles, profiler summaries, diagnostic rules, and related GradleMC data.

Reports are designed to improve troubleshooting, but they may include local paths, Java information, mod names, loader details, and runtime context. Review files before posting them publicly.

Local by design

No telemetry No analytics No cloud AI
No LLM integration No hidden online inference No broad private-file scanning

Smart Diagnostics is not generative AI. It is a local rule-based diagnostic system that evaluates available evidence, identifies missing information, calculates scores, and produces prioritized advice.

What GradleMC is not

  • Not a crash-fixing bot: it identifies useful signals and evidence; it cannot automatically repair every broken modpack.
  • Not a Gradle replacement: the name does not mean it replaces the Gradle build system.
  • Not a deep-profiler replacement: tools built for deep CPU, allocation, or tick profiling may still be required for advanced investigations.
  • Not cross-loader magic: Forge, Fabric, NeoForge, and Quilt jars are separate and are not interchangeable.
  • Not Bedrock software: GradleMC is built for supported Minecraft Java Edition releases only.

Frequently asked questions

Does GradleMC automatically fix lag or crashes?

No. It gathers evidence, highlights suspicious pressure signals, and makes the next troubleshooting step less blind.

Does Smart Diagnostics send anything online?

No. Smart Diagnostics and adaptive diagnostics operate locally and do not use cloud AI, an LLM, telemetry, or analytics.

Should I install GradleMC on the client or server?

Install it on the client for GUI, keybind, overlay, and FPS functionality. Install it on the server for server commands, reports, TPS/MSPT sampling, world-generation observation, and server-side diagnostics.

Can I use a Forge jar on NeoForge, Fabric, or Quilt?

No. Download the separate GradleMC file matching your exact loader and Minecraft version.

Where are my reports?

Look beneath <gameDir>/gradlemc/, or use /gradlemc reports latest.
