⚙️ GradleMC

In-game diagnostics, stability checks, and exportable troubleshooting reports for modded Minecraft.

Minecraft 1.20.1   •   Forge   •   Java 17   •   v1.0.0

GradleMC helps players, modpack makers, and testers inspect what is happening inside a modded Minecraft instance before troubleshooting turns into blind guessing, random mod deletion, and “it works on my machine” archaeology.

It provides a compact in-game GUI, readable commands, local stability checks, bounded performance tests, mod inspection tools, memory diagnostics, worldgen observation, and exportable reports for cleaner crash, lag, and support conversations.

✅ What is GradleMC?

GradleMC is a practical diagnostics companion for modded Minecraft. It does not automatically repair broken modpacks, replace crash-log reading, or perform deep profiling. Instead, it gives you organized local evidence so you can investigate problems with less guessing and more useful information.

Use it to check memory pressure, loaded mods, nearby entity load, block entity counts, server performance samples, client FPS snapshots, worldgen pressure, diagnostic reports, and issue bundles from inside the game.

✨ Key features

  • In-game dashboard: open GradleMC with /gradlemc gui or the configurable keybind.
  • Readable commands: use lowercase /gradlemc commands for status, memory, reports, mods, performance, entities, worldgen, and diagnostics.
  • Smart Diagnostics: view local stability scores, risk signals, explanations, and recommended next steps.
  • Adaptive Diagnostics: rule-based local checks that explain what they are seeing without pretending to be magic.
  • Memory inspection: review runtime memory usage and pressure indicators.
  • Mod inspection: count loaded mods, preview mod details, and search by mod ID or display name.
  • Entity scans: run bounded nearby entity scans from the player’s current position.
  • Block entity scans: inspect nearby block entity counts within a bounded radius.
  • Server performance sampling: collect short TPS/MSPT samples for a selected duration.
  • Client FPS testing: run bounded FPS tests and generate local FPS reports.
  • Worldgen observation: observe loaded chunk and worldgen-pressure signals during a short diagnostic session.
  • Reports and exports: generate local reports, diagnostics exports, and issue bundles for easier troubleshooting.

🧠 Adaptive Diagnostics, not cloud AI

GradleMC includes Adaptive Diagnostics. This means local heuristics, thresholds, status fields, and explainable recommendations based on diagnostic information collected during checks and short sampling sessions.

It is not ChatGPT, an LLM, generative AI, cloud AI, telemetry, embeddings, external inference, neural-network analysis, hidden analytics, or a background data collector.

  • No cloud AI.
  • No generative AI.
  • No telemetry.
  • No hidden analytics.
  • No automatic uploads.
  • No external inference.

🎮 How to open GradleMC

  • Run /gradlemc gui in-game.
  • Use the configurable Open GradleMC GUI keybind in the GradleMC controls category.
  • The default keybind is G.

⌨️ Useful commands

  • /gradlemc help - show command help.
  • /gradlemc gui - open the GradleMC GUI.
  • /gradlemc status - show a compact diagnostics overview.
  • /gradlemc memory - inspect memory status.
  • /gradlemc smart score - show the current local stability score.
  • /gradlemc smart advice - show prioritized recommendations.
  • /gradlemc smart explain - explain the current Smart Diagnostics result.
  • /gradlemc ai status - show Adaptive Diagnostics status through the compatibility command group.
  • /gradlemc mods count - count loaded mods.
  • /gradlemc mods search <text> - search loaded mods by ID or display name.
  • /gradlemc entities 128 - scan nearby entities within a bounded radius.
  • /gradlemc blockentities 128 - scan nearby block entities within a bounded radius.
  • /gradlemc perf start 30 - collect a 30-second server performance sample.
  • /gradlemc testfps start 30 - collect a 30-second client FPS sample.
  • /gradlemc worldgen start 10 - observe chunk and worldgen pressure for 10 seconds.
  • /gradlemc reports latest - show the latest report summary.
  • /gradlemc export - create a local diagnostics export.
  • /gradlemc issuebundle - create a troubleshooting bundle.

/gradlemc perf <seconds> is still available as a compatibility alias, but /gradlemc perf start <seconds> is the preferred command.

📦 Reports and exports

GradleMC writes local reports that help players, pack makers, and testers share useful troubleshooting context.

  • Diagnostics reports: summarize selected local checks and runtime context.
  • FPS reports: record bounded client FPS test results.
  • Performance reports: record short TPS/MSPT sampling sessions.
  • Worldgen reports: summarize passive chunk and worldgen-pressure observations.
  • Issue bundles: package troubleshooting evidence into a more shareable format.

Important: always review reports before sharing them publicly. They may contain useful technical context such as mod lists, Java details, local paths, runtime information, and diagnostic output.

  • Mod version: GradleMC v1.0.0.
  • Minecraft: Java Edition 1.20.1.
  • Loader: Forge.
  • Tested Forge version: 47.4.20.
  • Java: 17.
  • Mod ID: gradlemc.
  • Release file: gradlemc-1.0.0-forge-1.20.1.jar.

This v1.0.0 release is for Forge 1.20.1. It does not claim support for Fabric, Quilt, NeoForge, Bedrock Edition, or other Minecraft versions.

GUI, keybind, and FPS testing are client-side features. GradleMC is designed with client-only features isolated where needed, but dedicated-server usage should still be tested in your own setup before relying on it for a public server pack.

🚀 Installation

  1. Install Minecraft Forge for Minecraft Java Edition 1.20.1.
  2. Use Java 17.
  3. Place gradlemc-1.0.0-forge-1.20.1.jar in your instance mods folder.
  4. Launch the game.
  5. Run /gradlemc gui or press the configured keybind.

⚠️ Known limitations

  • GradleMC does not automatically fix broken modpacks.
  • GradleMC does not replace crash-log reading, Spark, VisualVM, or deeper profiling tools.
  • Short FPS, TPS, MSPT, and worldgen samples are snapshots, not long-term benchmarks.
  • Worldgen observation is passive and does not force-generate chunks.
  • Adaptive Diagnostics uses local explainable heuristics, not cloud AI or generative AI.
  • Reports organize evidence, but they cannot prove every possible compatibility issue.
